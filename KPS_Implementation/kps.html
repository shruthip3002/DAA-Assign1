Time Complexity Analysis

The time complexity of the Kirkpatrick-Seidel (KPS) algorithm, which is used to compute the convex hull of a set of points in two dimensions, is \(O(n \log h)\), where \(n\) is the number of input points and \(h\) is the number of points on the convex hull.

Here's a breakdown of the time complexity:

  1. Initial Sorting:
    • Sorting the input points initially takes \(O(n \log n)\) time using an efficient sorting algorithm like quicksort or mergesort.
  2. Divide-and-Conquer Phase:
    • The divide-and-conquer phase of the KPS algorithm recurses on the left and right subsets of the input points.
    • Each recursion level splits the set of points in half, resulting in \(O(\log n)\) levels.
    • At each level, finding the upper and lower bridges takes linear time, which is \(O(n)\).
    • Thus, the total time spent in this phase is \(O(n \log n)\).
  3. Merging Phase:
    • After finding the upper and lower bridges, merging them together to form the convex hull takes linear time, which is \(O(n)\).

Overall, the time complexity of the KPS algorithm is dominated by the divide-and-conquer phase, resulting in a total time complexity of \(O(n \log h)\), where \(h\) is the number of points on the convex hull. Since \(h \leq n\), in the worst case scenario where all points are on the convex hull, the time complexity simplifies to \(O(n \log n)\).
